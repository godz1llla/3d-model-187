# Решение проблемы с 3D моделями

## Проблема
3D модели не загружаются, потому что GLTFLoader из CDN может не работать правильно.

## Решение

### Вариант 1: Скачать GLTFLoader локально (РЕКОМЕНДУЕТСЯ)

1. Откройте в браузере: https://unpkg.com/three@0.128.0/examples/js/loaders/GLTFLoader.js
2. Сохраните содержимое в файл `js/GLTFLoader.js`
3. В файле `object.html` замените строку:
   ```html
   <script src="https://unpkg.com/three@0.128.0/examples/js/loaders/GLTFLoader.js"></script>
   ```
   на:
   ```html
   <script src="js/GLTFLoader.js"></script>
   ```

### Вариант 2: Проверить консоль браузера

1. Откройте сайт через локальный сервер
2. Нажмите F12 для открытия консоли разработчика
3. Перейдите на вкладку "Console"
4. Посмотрите, какие ошибки там есть
5. Если видите ошибку про GLTFLoader - значит нужно скачать его локально

### Вариант 3: Использовать другой CDN

В файле `object.html` можно попробовать заменить CDN на:
```html
<script src="https://threejs.org/examples/js/loaders/GLTFLoader.js"></script>
```

## Проверка работы

После исправления:
1. Перезапустите локальный сервер
2. Откройте страницу объекта (например: http://localhost:8000/object.html?id=1)
3. 3D модель должна загрузиться и отображаться

## Если проблема сохраняется

1. Убедитесь, что файлы .glb находятся в папке `models/`
2. Убедитесь, что имена файлов точно совпадают с именами в массиве OBJECTS в `js/main.js`
3. Проверьте консоль браузера (F12) на наличие ошибок
4. Убедитесь, что используете локальный сервер (не открываете файл напрямую)
